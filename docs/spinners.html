Sadrazam – Spinner Showcase

Loading indicator variants, reference counting, and placement options

Main Spinner (Full-page)

Fixed center overlay spinner. Shown via Sadrazam.Spinner.show() or Sadrazam.Spinner.show({ type: 'main' }) and hidden with Sadrazam.Spinner.hide().

Show / Hide .spinner-main

The main spinner appears at the center of the viewport with a high z-index. It auto-hides after 2 seconds in this demo.

Helper Spinner (Background tasks)

Fixed bottom-left spinner for background operations. Triggered with Sadrazam.Spinner.show({ type: 'helper' }).

Show / Hide .spinner-helper

The helper spinner sits at the bottom-left corner, ideal for non-blocking background tasks. Auto-hides after 2 seconds in this demo.

Reference Counting

Multiple async operations can call show() independently. The spinner only disappears when every show() has been matched by a hide(). The internal counter prevents premature removal.

Click "3 overlapping requests" to simulate three async calls that start together but finish at different times (1s, 2s, 3s). The spinner stays visible until the last one completes.

Block Spinner (CSS-only)

An in-flow spinner for indicating a specific region is loading. Applied directly via .spinner-block on a <div>. No JS required.

Default <div class="spinner-block"></div>

The block spinner is relatively positioned and centered with auto margins. Ideal for card or panel loading states.

Inline Spinner (CSS-only)

Sits alongside text or inside buttons. Inherits the current font size via 1em sizing. Applied via .spinner-inline. No JS required.

API Reference

Complete static API for Sadrazam.Spinner

show() Displays the spinner. Increments the internal reference counter.
Sadrazam.Spinner.show() // type defaults to 'main'
Sadrazam.Spinner.show({ type: 'helper' })
hide() Decrements the reference counter. Removes spinner from DOM when counter reaches 0.
Sadrazam.Spinner.hide()
help() Prints available configuration options to the browser console.
Sadrazam.Spinner.help()
